Este documento descreve toda a arquitetura, configuração e funcionamento do sistema de sincronização automática de dados entre as 38 lojas e o servidor central.

1. Visão Geral do Sistema

O sistema sincroniza dados do Linx DataSync (Retail) entre o servidor central e cada uma das 38 lojas, em dois sentidos:

OperaçãoDireçãoO que faz
RECEBELoja → ServidorColeta dados de vendas, clientes e movimentos de cada loja
ENVIAServidor → LojaEnvia tabelas, produtos, preços e parâmetros atualizados para cada loja

Fluxo de um ciclo completo

1
RECEBE — todas as 38 lojas em lotes de 5 simultâneas (timeout: 2 min por loja)
2
Pausa de 15 minutos — aguarda o Linx processar os dados recebidos
3
ENVIA — apenas as lojas que tiveram RECEBE com sucesso, em lotes de 5 (timeout: 8 min por loja). Lojas com RECEBE com falha são ignoradas no ENVIA.
Por que ignorar o ENVIA quando o RECEBE falha?
Se a loja não conseguiu enviar dados (RECEBE), é porque está sem conexão ou desligada. Não adianta tentar enviar dados para ela.

5. Como Fazer Deploy de Alterações

Edite os arquivos locais em C:\Users\Daniella\ti\ e execute no PowerShell local:

$cred = New-Object System.Management.Automation.PSCredential('Datasync',
    (ConvertTo-SecureString 'suporte2003' -AsPlainText -Force))
$s = New-PSSession -ComputerName 192.168.0.147 -Credential $cred

# Enviar script principal
$txt = [System.IO.File]::ReadAllText("C:\Users\Daniella\ti\data-sync-automacao.ps1",
    [System.Text.Encoding]::UTF8)
Invoke-Command -Session $s -ScriptBlock {
    param($t)
    [System.IO.File]::WriteAllText(
        "C:\Users\Datasync\Desktop\ti\data-sync-automacao.ps1", $t,
        [System.Text.Encoding]::UTF8)
} -ArgumentList $txt

# Enviar e reiniciar painel
$txt2 = [System.IO.File]::ReadAllText("C:\Users\Daniella\ti\gerar-painel-datasync.ps1",
    [System.Text.Encoding]::UTF8)
Invoke-Command -Session $s -ScriptBlock {
    param($t)
    [System.IO.File]::WriteAllText(
        "C:\Users\Datasync\Desktop\ti\gerar-painel-datasync.ps1", $t,
        [System.Text.Encoding]::UTF8)
    Stop-ScheduledTask -TaskName "DataSyncPainel"
    Start-Sleep -Seconds 2
    Start-ScheduledTask -TaskName "DataSyncPainel"
} -ArgumentList $txt2

Remove-PSSession $s
Sempre verifique a sintaxe antes de fazer deploy:

6. Configurações Atuais do Script Principal

ParâmetroValorO que é
$loteTamanho5Lojas processadas em paralelo por vez
Timeout RECEBE2 minutos (120.000ms)Tempo máximo para operação de recebimento por loja
Timeout ENVIA8 minutos (480.000ms)Tempo máximo para operação de envio por loja
Pausa entre fases15 minutos (900s)Aguarda o Linx processar antes do ENVIA
Por que lotes de 5? Rodar todas as 38 lojas ao mesmo tempo sobrecarrega o Linx DataSync e causa falhas em cascata. Lotes de 5 equilibram velocidade e estabilidade do sistema.
Por que timeout de 8 min no ENVIA? Lojas de shopping têm alto volume de dados (2.000–3.000+ registros de clientes, entradas, NFC-e). Com 3 minutos, o processo era morto no meio da transferência.

Detecção de sucesso

Após cada operação, o script lê a última linha do log gerado pelo Linx DataSync. Sucesso é confirmado quando a linha contém:

transferência foi concluída com sucesso

Arquivos de status em tempo real

Cada job paralelo escreve seu estado em C:\Logs\DataSync\status\loja_XX.txt:

TIPO|STATUS|HH:MM:SS

Exemplos:
RECEBE|RODANDO|10:31:05
RECEBE|OK|10:32:18
ENVIA|IGNORADO|10:45:00
ENVIA|ERRO|10:53:12

8. Interpretando Falhas

Mensagem no logCausa provávelO que fazer
TIMEOUT apos X minutosLoja demorou mais que o timeout (dados volumosos ou conexão lenta)Verificar no ciclo seguinte; se persistir, aumentar timeout no script
Log do Linx nao geradoLinx não iniciou a transferência — loja offline ou atalho inválidoVerificar conectividade da loja e o arquivo .lnk

10. Decisões Técnicas Relevantes

DecisãoMotivo
Lotes de 5 em paralelo38 simultâneas sobrecarrega o Linx e causa falhas em cascata. 5 é o equilíbrio entre velocidade e estabilidade.
RECEBE → pausa → ENVIA só das OKLoja sem RECEBE está offline. A pausa garante que o Linx processe os dados antes do envio.
Timeout ENVIA de 8 minutosLojas de shopping têm 2.000–3.000+ registros. Com 3 min o processo era morto no meio da transferência.
Arquivo de status por lojaJobs paralelos têm escopo isolado. Arquivo é a única forma de comunicar status em tempo real ao painel.
Deploy via [System.IO.File]::WriteAllTextGarante UTF-8 sem BOM. PSSession com Set-Content pode gerar encoding incorreto.
Regra de firewall porta 8080Sem a regra inbound, o painel não abre em outros computadores da rede, mesmo com o servidor HTTP rodando.
